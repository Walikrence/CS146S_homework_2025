## 探索初始应用
一个极简的全栈初始应用。
- FastAPI 后端 + SQLite（SQLAlchemy）
- 静态前端（无需 Node 工具链）
- 基础测试（pytest）
- Pre-commit 钩子（black + ruff）
- 用于练习代理驱动工作流的任务

以此应用作为你的实验场，测试和运行你构建的 Warp 自动化工具。

### 目录结构

```
backend/                # FastAPI 应用
frontend/               # 由 FastAPI 提供服务的静态前端
data/                   # SQLite 数据库 + 种子数据
docs/                   # 代理驱动工作流的任务文档
```

### 快速开始

1) 激活 conda 环境。

```bash
conda activate cs146s
```

2) （可选）安装 pre-commit 钩子

```bash
pre-commit install
```

3) 运行应用（在 `week5/` 目录下）

```bash
make run
```

4) 打开 `http://localhost:8000` 访问前端界面，打开 `http://localhost:8000/docs` 查看 API 文档。

5) 体验初始应用，熟悉其现有功能。


### 测试
运行测试（在 `week5/` 目录下）
```bash
make test
```

### 代码格式化 / 静态检查
```bash
make format
make lint
```

## 第一部分：构建自动化工具（至少选择 2 项）
从 `week5/docs/TASKS.md` 中选择任务来实现。你的实现必须以下面两种方式使用 Warp（详情见下文）：

- A) 使用 Warp Drive 功能 —— 如保存的提示词、规则或 MCP 服务器。
- B) 在 Warp 中构建多代理工作流。

将你的更改限制在 `week5/` 中的后端、前端、逻辑或测试部分。
对于每个选定的任务，请注明其难度级别。


### A) Warp Drive 保存的提示词、规则、MCP 服务器（必需：至少一项）
创建一个或多个可共享的 Warp Drive 提示词、规则或 MCP 服务器集成，针对本仓库进行定制。示例：
- 带覆盖率和失败测试重试的测试运行器
- 文档同步：从 `/openapi.json` 生成/更新 `docs/API.md`，列出路由变更
- 重构辅助：重命名模块，更新导入，运行静态检查/测试
- 发布助手：版本号升级，运行检查，准备变更日志片段
- 集成 Git MCP 服务器，让 Warp 自主与 Git 交互（创建分支、提交、PR 说明等）

>*提示：保持工作流聚焦、传递参数、使其幂等、尽量优先使用无头/非交互式步骤。*

### B) Warp 中的多代理工作流（必需：至少一项）
运行多代理会话，在不同的 Warp 标签页中让独立的代理同时处理不同任务。
- 在不同的 Warp 标签页中使用并行代理执行 `TASKS.md` 中的多个独立任务。挑战：你能同时运行多少个代理？

>*提示：[git worktree](https://git-scm.com/docs/git-worktree) 在这里可能很有帮助，可以防止代理之间互相干扰。*
