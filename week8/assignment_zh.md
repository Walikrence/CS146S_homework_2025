## 作业概述
使用 3 种不同的技术栈构建相同的功能性 Web 应用。至少一个版本必须使用 [`bolt.new`](https://bolt.new/)（一个 AI 应用生成平台）创建。至少一个版本的前端或后端必须使用非 JavaScript 语言（例如 Django、Ruby on Rails）。

你可以复用之前几周的应用（"开发者命令中心"），也可以创建一个新应用，只要满足[最低功能范围](#最低功能范围)即可。应用应具备端到端的完整功能（前端 + 后端 + 适当的持久化存储），并展示一组连贯的功能集。

## 最低功能范围
- 用户可以对主要资源（如笔记、任务、帖子等）进行创建、读取、更新和删除操作。
- 在技术栈适用的情况下使用持久化存储（数据库或文件存储）。
- 基本的验证和错误处理。
- 简洁但功能完整的 UI，能够展示主要操作流程。
- 每个版本的本地运行说明要清晰明了（如已部署则附上部署链接）。

## 技术栈要求
构建同一应用的 3 个独立版本，每个版本使用不同的技术栈。示例：
- MERN（MongoDB、Express、React、Node.js）
- MEVN（MongoDB、Express、Vue.js、Node.js）
- Django + React（或 Vue）
- Flask + 原生 JS（或 React）
- Next.js + Node（或 NestJS）
- Ruby on Rails（全栈）

提醒：至少一个版本的前端或后端必须包含非 JavaScript 语言（例如 Python/Django、Ruby/Rails）。


至少一个版本必须使用 AI 应用生成平台 **[`bolt.new`](https://bolt.new/)** 构建，但其他版本你可以自由探索其他应用生成平台（如 Lovable、Figma Make）。


## 了解 Bolt
Bolt 是一个 AI 辅助开发平台，能够根据自然语言提示生成网站、Web 应用和移动应用。用户可以用纯文本描述自己的想法，Bolt 会在几分钟内生成一个可运行的原型——从着陆页和电商网站到 CRM 和移动工具。点击[此处](https://support.bolt.new/building/intro-bolt)了解更多。

## AI 应用生成器使用提示
- Bolt 等应用生成器最适合现代全栈技术，在不指定特定框架时默认使用这些技术。
- 建议从一个干净的提示开始，描述你的应用概念、数据实体、路由和 UI 流程。
- 在提示中清晰描述数据模型和关联关系。
- 通过迭代优化提示来完善数据模型、CRUD 端点、认证（如使用）和前端组件。
- 保持每个版本独立，以避免依赖冲突。
- 导出或同步生成的代码，并将其作为该技术栈的独立项目文件夹提交。

## 交付物
