# 第六周 — 使用 Semgrep 扫描和修复漏洞

## 作业概述
使用 **Semgrep** 对 `week6/` 中提供的应用运行静态分析。筛选扫描结果并修复至少 3 个安全问题。在报告中说明 Semgrep 发现了哪些问题以及你是如何修复的。


## 了解 Semgrep
Semgrep 是一款开源的静态分析工具，用于搜索代码、发现缺陷，并实施安全防护措施和编码规范。

1. 点击[此处](https://github.com/semgrep/semgrep/blob/develop/README.md)了解 Semgrep。

2. 按照上述链接中的安装说明进行安装。你可以选择使用 **Semgrep Appsec Platform** 或 **CLI 工具**。


## 扫描任务

### 扫描对象
- 后端 Python（FastAPI）：`week6/backend/`
- 前端 JavaScript：`week6/frontend/`
- 依赖项：`week6/requirements.txt`
- 配置/环境文件（检查密钥泄露）：`week6/` 下的文件


### 运行通用安全扫描以及针对密钥和依赖的专项扫描。

从**作业仓库根目录**执行以下命令，应用精选的 CI 风格规则包（包含代码规则和密钥规则）：
```bash
semgrep ci --subdir week6
```

## 任务
1. 从 Semgrep 识别出的问题中选择任意 3 个，使用你选择的 AI 编程工具进行修复。

2. 展示精确的修改内容并解释缓解措施（例如参数化 SQL、更安全的 API、更强的加密、清洁化 DOM 写入、限制 CORS、升级依赖等）。

3. 重要：确保修复后应用仍可正常运行且测试仍能通过。

## 交付物
### 1. 发现概述
- 总结 Semgrep 报告的各类问题（SAST / 密钥 / SCA）。
- 记录你选择忽略的误报或噪声规则及其原因。

### 2. 三项修复（修复前 → 修复后）
对于每个修复的问题：
- 文件和行号
- Semgrep 标记的规则/类别
- 简要风险描述
- 你的修改内容（简短的代码差异或说明，AI 编程工具使用情况）
- 为什么此修改能缓解该问题


## 提示
- 优先进行最小化、有针对性的修改，解决根本原因。
- 每次修复后重新运行 Semgrep，确认发现已消除且未引入新问题。
- 对于依赖项，记录升级的版本号，如使用了供应链扫描请附上安全公告链接。
